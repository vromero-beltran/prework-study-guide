HTML

  • You must always start the VS Code with the Declaration, !DOCTYPE html in between the tags to let the browser know to expect HTML on the page.
  • The next line of code should always be the representation of the start of the documentation and the language, html lang="en" between the tags.
  • The head element contains information about the webpage.
  • The most important head element is the metadada element, that helps the browser render the page correctly.
  • The body element represents the visible content shown to the user.
  • The h1 element is the header for the body element and the number 1 should only be used onced in each website.
  • The p element is the paragraph section and can be placed anywhere after the header.
  • Other elements used in the body element are the ul, ol, li, img, br, and a.
  • The ul element stands for unordered list and is used to make lists in bullet points.
  • The ol element stands for ordered list and is used to make lists in numerical order.
  • The li element stands for list item and is used in both ul, and ol elements to start making the list.
  • The img element contains information about a image and must include the src attribute equaling the URL and the alt attribute equaling the title of the URL.
  • The br element is a line of empty space to create a break for the next section usaully used after a picture.
  • The a element stands for archer and is used to add a link in the page with the href attribute equaling the URL.
  • The footer element is used after the body element and it contains the author, copyright, contacts, site map, and navigations.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
